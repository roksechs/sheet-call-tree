# 出力フォーマット

`sheet-call-tree` は `--ref-mode` で選択できる 4 つの描画モードをサポートします。このモードは**数式セル参照**が出力にどのように現れるかを制御します。定数セル参照（数式ではなく通常の値を含むセル）はモードに関わらず常にスカラー値に解決されます。

全ての例で同じワークブックを使用します：

| セル | 値 |
|------|-----|
| A1 | `10` |
| A2 | `20` |
| C5 | `=SUM(A1:A2)` |
| B10 | `=C5+1.1` |
| B11 | `=C5*2` |

---

## `ref`（デフォルト）

**使用場面：** 各数式セル参照がナビゲート可能なクロスリファレンス文字列として表示される、コンパクトな依存関係マップが欲しいとき。人間によるレビューや差分に適しています。

数式セル参照は `'@シート名!セル参照'` 文字列として表示されます。同じ YAML ドキュメント内の別のトップレベルキーを参照として追うことができます。定数セル参照（A1=10、A2=20）はインラインでスカラー値に解決されます。

```bash
sheet-call-tree example.xlsx
```

```yaml
Sheet1!B10:
  ADD:
  - '@Sheet1!C5'
  - 1.1
Sheet1!B11:
  MUL:
  - '@Sheet1!C5'
  - 2
Sheet1!C5:
  SUM:
  - RANGE:
    - 10
    - 20
```

`Sheet1!B10` は `ADD` を 2 つの引数で呼び出しています：数式セル `Sheet1!C5` への参照と定数 `1.1` です。`Sheet1!C5` はフル AST を持つ独立したトップレベルエントリとして現れます。

---

## `ast`

**使用場面：** クロスリファレンスを追わずに完全な依存関係ツリーをその場で展開したいとき。詳細分析や後続ツールへの入力に便利です。

数式セル参照は単一キーのマッピング `{'@シート名!セル参照': <サブツリー>}` として表示され、参照先セルの AST をインラインに埋め込みます。複数のセルが同じ数式セルに依存している場合、同じサブツリーが複数回現れることがあります。

```bash
sheet-call-tree example.xlsx --ref-mode ast
```

```yaml
Sheet1!B10:
  ADD:
  - '@Sheet1!C5':
      SUM:
      - RANGE:
        - 10
        - 20
  - 1.1
Sheet1!B11:
  MUL:
  - '@Sheet1!C5':
      SUM:
      - RANGE:
        - 10
        - 20
  - 2
Sheet1!C5:
  SUM:
  - RANGE:
    - 10
    - 20
```

`@Sheet1!C5` キーはフルの `SUM(RANGE(10, 20))` サブツリーを値として持ち、別のエントリにジャンプすることなく `Sheet1!B10` と `Sheet1!B11` の中で直接確認できます。

---

## `value`

**使用場面：** AST 構造ではなく、Excel がキャッシュした各数式セル参照の計算結果を確認したいとき。結果のスポットチェックに便利です。

数式セル参照はワークブックの `data_only` 表現に格納されたキャッシュ済みスカラー値として表示されます。ワークブックがプログラムで作成され（例: openpyxl を使用）、Excel で一度も開かれていない場合、計算値はキャッシュされておらず `null` として表示されます。

```bash
sheet-call-tree example.xlsx --ref-mode value
```

```yaml
Sheet1!B10:
  ADD:
  - null
  - 1.1
Sheet1!B11:
  MUL:
  - null
  - 2
Sheet1!C5:
  SUM:
  - RANGE:
    - 10
    - 20
```

`Sheet1!C5` 参照の `null` 値は、サンプルワークブックがプログラムで作成されており、キャッシュ済みの数式結果がないために現れます。Excel で保存されたワークブックでは、ここに実際のスカラー値が入ります。

---

## `inline`

**使用場面：** 最もコンパクトな人間が読める表現が欲しいとき、またはコメントやチケット、ログへの貼り付け用に 1 セルあたり単一の式文字列が必要なとき。

各数式セルの依存関係ツリー全体が単一の `FUNC(arg1, arg2, …)` 式文字列として表示されます。参照された数式セルは再帰的に展開されます。結果は セル参照 → 式文字列 のフラットな YAML マッピングです。

```bash
sheet-call-tree example.xlsx --ref-mode inline
```

```yaml
Sheet1!B10: ADD(SUM(RANGE(10, 20)), 1.1)
Sheet1!B11: MUL(SUM(RANGE(10, 20)), 2)
Sheet1!C5: SUM(RANGE(10, 20))
```

`Sheet1!C5` は `Sheet1!B10` と `Sheet1!B11` に完全にインライン展開されていることに注意してください — `ADD(SUM(RANGE(10, 20)), 1.1)` — 名前で参照されるのではなく直接埋め込まれています。

---

## 全モードにおける定数セル参照

セルの値が通常のスカラー（数値、文字列、真偽値）であり数式でない場合、そのセルは*定数セル*です。4 つのモード全てにおいて、定数セル参照は直接スカラー値に解決されます。上の例では、`A1=10` と `A2=20` は `--ref-mode` に関わらず、常に `RANGE` ノードの中で `10` と `20` として現れます。
